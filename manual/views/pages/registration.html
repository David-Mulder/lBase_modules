Registration

Registration allows the use of shorthands to access libraries or modules

Register an object

To access a library you would normally use 

<cfset this.app.lib("example").someFunction()>

However this is relatively long and because of this registration was introduced. Registration allows you to access the same object like this 

<cfset this.example.someFunction()>

or this 

<cfset this.app.example.someFunction()>

To get this to work you first need to register the module. Registration is simply done by calling 

<cfset this.register(this.app.lib("example"))>

Autoregistration

If you want an object to be globally accessible it's possible to autoregister libraries and modules from config/base.cfc and adding 

<cfset config["autoregister"] = {}>
<cfset config["autoregister"]["libraries"] = ["modules.core.console","example"]>
<cfset config["autoregister"]["modules"] = ["core","template"]>

From the application it's possible to autoregister module libraries, from inside a module only modules and it's own libraries can be autoloaded.

Please also note that from a module it's still possible to access modules that are autoregistered in the application through 

<cfset this.app.core.console.log()>

However this should only be done with modules that are ALWAYS autoregistered (such as the core module)
